# Contributing Guidelines

## Branching
- `main` → production
- `dev` → development
- `feature/*` → features
- `hotfix/*` → urgent fixes

## Pull Requests (PRs)
- PRs must target `dev` first.
- CI checks must pass before merging.
- Assign at least one reviewer from dev or QA leads.

## Code Standards
- Follow coding style guides.
- No direct push to `main`.
- Keep secrets out of code.
